Form Element

Tag Name Description Example
<input/> Input Element Use to create input controls to accept data from the user. There are a vase number of types and attributes that are associated with the input tag; here we are discussing 8 common variations.
The name attribute and the type attribute are normally required for the element to run. 		< input name = "food" type = "checkbox" value = "lasagna"/>
Name Attribute
The name attribute is necessary to submit a form. In the examples column, the input with the missing name attribute will not be able to submit it's information.		 < input name = "applepie" id = "dessert"/>

< input id = "dessert"/>
Checkbox Type
The checkbox type creates a single checkbox item. Multiple boxes can be created but need to share the same name atttibute for the input to work. Users can select multiple checkboxes at once.		 < input type = "checkbox" name="dessert" value="applepie"/>

< input type = "checkbox" name = "dessert" value = "chocolate chip cookie"/>

< input type = "checkbox" name = "dessert" value = "vanilla icecream"/>
Number Type
The number type allows users to type only numbers and a few special characters.		 < input type = "number" name = "expiration date" />
Range Type
The range type allows users to create a slider between a minimum and maximum value. The purpose of this type is more to create a visual widget.		 < input type = "range" name = "Sweetness rating" min = "1" max = "10" step = "1" />
Text Type
The text type displays a single blank line for inputting text.		 < input type = "text" name = "Favorite Autumn Fruit" />
Radio Button Type
The radio type allows users to create a radio button. Multiple radio buttons can be create but all must share the same name attribute. Unlike the checkbox type, only one radio button can be selected at a time. 		< input type = "radio" name = "Type of cheese" value = "Colby" />

< input type = "radio" name = "Type of cheese" value = "Cheddar" />

< input type = "radio" name = "Type of cheese" value = "Swiss" />
Password Type
The password type displays a single line where the input information is censored. This type is used for inputting sensitive information. When submitting, the value is sent uncensored.		 < input type = "text" name = "VegHead username" />

< input type = "password" name = "password" />
Submit Type
The submit type displays a button that will execute an action (submit the form). By default, the text on the button reads 'submit'. 		< input type = "submit" value = "send" />
<form></form> Form Used for collecting and sending information to some source (whether it be external or within the same folder).
Paired with the action attribute and the method attribute. 		<form method="post" action="index.html"> Enter favorite comfort food:
<input type="text" name="food"/>
<input type="submit"/></form>
<label></label> Label Provides an ID for an input element; Almost always paired with the id attribute and the for attribute. <label for="food"> Name of food: </label>
<select></select> Select Used to create a dropdown list. Paired with the name attribute and the value attribute as well as the option element. <select name="types of breads"> <option value="potato"> Potato </option> <option value="wheat"> Wheat </option>
<option value="white"> White </option>
</select>
<datalist></datalist> Datalist Creates an autocomplete/select function for a webpage.
Commonly paired with the input element, the option element, the id attribute, and the value attribute.		 <input list="apples"/> <datalist id="apples"> <option value="fuji"/><option value="granny smith"/> <option value="red delicious"/> </datalist>
<textarea></textarea> Textarea Used to create a form with multiple lines for input.
Commonly paired with the rows attribute, the cols attribute, and the name attribute. However, this is not a exhausted list; other attributes can be added if needed.		 <textarea cols="5" row="6" name="recipe"> </textarea>
